PART A · 1 — Why study RNA structure?

DNA holds information passively — it just sits in the double helix and waits to be read. RNA, however, folds into a 3-D shape, and that shape is its function. A tRNA's cloverleaf shape is what lets it carry an amino acid; a ribosome's complex RNA scaffold is what catalyzes peptide bond formation; a microRNA's hairpin is what targets it for processing.

The central problem

RNA is synthesized as a single strand. Inside this single strand, complementary bases find each other and pair up, folding the molecule onto itself. Given just the nucleotide sequence (A, C, G, U), can we predict which bases will pair with which? That folded base-pairing pattern is the secondary structure, and predicting it is one of the foundational problems of bioinformatics.

From RNA_structure.pdf:

  • RNA sequence analysis is different from DNA — DNA usually exists as a stable duplex, but RNA folds.
  • For RNA, structure conservation is more important than sequence conservation: across evolution, the bases at a paired position can change as long as the pair is preserved (so AU → GC happens often). This is the whole basis of covariance methods.

PART A · 2 — RNA Basics

Composition

RNA is a polymer of nucleotides built from four bases: Adenine, Cytosine, Guanine, and Uracil (replaces Thymine of DNA). The sugar is ribose (DNA has deoxyribose — the difference is an extra –OH at the 2′ position of ribose).

Base pairs in RNA

PairType# H-bondsStability
G ≡ CCanonical Watson-Crick3Most stable
A = UCanonical Watson-Crick2Less stable than GC
G · UNon-canonical "wobble"2Allowed and biologically important

Wobble pairing — exam essential

The G–U wobble is the most important non-canonical pair. It is geometrically allowed and is regularly used in algorithms (and on exam questions — the Comprehensive paper explicitly asked you to use G:U wobble in Nussinov).

Rule of thumb: each base pairs with at most ONE other base. No base triples in elementary RNA models.

RNAs are biological multi-tools:

PART A · 3 — RNA Secondary Structure Elements

When an RNA strand folds back on itself, the resulting shapes have names you must know cold:

Hairpin (stem-loop)A stem capped by an unpaired loop. Loop is generally at least 4 bases long.
Bulge loopUnpaired bases on only one side of a stem

PseudoknotBases inside one loop pair with bases outside that loop — produces crossing base pairs
Kissing hairpinsTwo separate hairpin loops base-pair with each other

⚠ The pseudoknot constraint — critical for Nussinov

A pseudoknot arises when base pairs cross: that is, you have pairs (i, j) and (i′, j′) with i < i′ < j < j′. Standard Nussinov and Zuker dynamic programming algorithms CANNOT handle pseudoknots — that case is explicitly prohibited by the recursion (the bifurcation rule splits a region into two independent halves, which only nested structures allow).

PART A · 4 — Representing RNA Secondary Structure

1) The classic 2-D drawing

Lines for backbone, dashes for hydrogen bonds. Easy to read for small molecules; messy for big ones.

2) Dot-bracket (Vienna) notation

One character per nucleotide:

  • . = unpaired
  • ( = paired with a downstream base
  • ) = paired with an upstream base — matching parentheses identify the pair

Example: .(((...))) means position 1 is unpaired; positions 2-4 each pair with positions 10-8; positions 5-7 form the unpaired loop.

3) Circular representation (the diagram the Comprehensive PYP asks about)

From RNA_structure.pdf, this is defined precisely:

  • Place the bases on the circumference of a circle, in sequence order.
  • For every base pair, draw a chord (arc) between the two paired bases.
  • If any two arcs cross, a pseudoknot is present.

🎯 PYP Q1(d) — circular representation & Nussinov capability

The Compre paper shows a 49-base RNA and asks you to (a) draw its circular representation and (b) assess whether Nussinov can predict it. Your answer should follow this two-step logic:

  1. Draw arcs. Bases on the circle, one arc per H-bond.
  2. Test crossings.
    • If no arcs cross → structure is nested (no pseudoknot) → Nussinov can in principle recover it, but Nussinov simply maximizes the number of pairs and ignores thermodynamics, nearest-neighbor stacking, and loop-length preferences — so it may find a structure with the same or more pairs that isn't the biological one.
    • If any arcs cross → pseudoknot present → Nussinov cannot predict this structure, because its bifurcation rule only generates nested structures.

PART A · 6 — ⭐ Nussinov Algorithm (deep dive)

The problem Ruth Nussinov solved (1978): Given an RNA sequence, find the secondary structure (set of non-crossing base pairs) that contains the maximum number of base pairs.

Why this works as a starting point: more base pairs ≈ more hydrogen bonds ≈ generally more stable. It's a crude approximation of thermodynamics — but it has the redeeming property that it can be solved exactly and efficiently using dynamic programming.

6.1 — The rules (verbatim from the slides)

String uses {A, U, G, C}. Pairs allowed: A–U, G–C, and G–U (wobble). The structure must satisfy:

  1. i < j − 4 (minimum loop separation — usually a hairpin loop needs ≥ 3 unpaired bases between paired positions; you can change this choice). "You can change your choice" means problem statements may relax this to j − i ≥ 2 or any other value.
  2. Only A–U, G–C, and G–U can pair.
  3. "Pairing forms a matching" — each position pairs with at most one other.
  4. If (i, j) and (i′, j′) are paired, then the arrangement i < i′ < j < j′ is prohibited — i.e., no crossing pairs / no pseudoknots.

6.2 — Core idea of the recursion

Define γ(i, j) = maximum number of base pairs in the optimal folding of the subsequence xi…xj.

The slide gives the key insight: there are exactly four ways to obtain the best structure for (i, j) from smaller subsequences:

γ(i, j) = max of these four cases:

① γ(i+1, j) — position i is unpaired
② γ(i, j−1) — position j is unpaired
③ γ(i+1, j−1) + δ(i, j) — i pairs with j (δ = 1 if pair allowed, else 0)
④ maxi < k < j [ γ(i, k) + γ(k+1, j) ] — bifurcation: split into two independent sub-structures

Why these four exhaust every possibility: In any optimal structure of (i, j), position i is either (a) unpaired → case ①, (b) paired with j → case ③, or (c) paired with some interior k, which splits the region into (i, k) and (k+1, j) → case ④. Similarly considering j gives case ②. Together these cover every legal nested structure.

6.3 — Initialization (the base case)

γ(i, i) = 0   for i = 1 to L
γ(i, i−1) = 0   for i = 2 to L

Diagonal cells are zero (an interval of length 1 has no pairs), and the "below diagonal" sentinel cells are zero too (an empty interval has no pairs). All other cells start with no value and are filled in by the recursion.

6.4 — Fill order: how to populate the DP matrix

Because γ(i, j) depends on γ(i+1, j), γ(i, j−1), γ(i+1, j−1), and γ(i, k) + γ(k+1, j), you fill the matrix diagonal by diagonal, starting with the shortest subsequences (length 2) and working up to the full sequence (length L).

  • Compute all cells with j − i = 1, then j − i = 2, then j − i = 3 … up to j − i = L − 1.
  • The answer to "max pairs in the whole sequence" is γ(1, L) (top-right corner).

6.5 — Traceback (the second stage)

γ(1, L) gives the maximum number of pairs, but not which pairs. To recover the structure, we trace back through the DP table using a stack:

Initialization: push (1, L) onto stack.

Recursion: repeat until stack is empty:
  1. Pop (i, j).
  2. If i ≥ j: continue (nothing to do).
  3. Else if γ(i+1, j) == γ(i, j): push (i+1, j). i is unpaired
  4. Else if γ(i, j−1) == γ(i, j): push (i, j−1). j is unpaired
  5. Else if γ(i+1, j−1) + δ(i, j) == γ(i, j):
    • Record (i, j) as a base pair
    • Push (i+1, j−1).
  6. Else for k = i+1 to j−1, find the k where γ(i, k) + γ(k+1, j) == γ(i, j):
    • Push (k+1, j) and (i, k).
    • Break out of the for-loop.

About ties

There may be multiple optimal structures with the same maximum number of pairs. The traceback rules are checked in order, so the algorithm will return ONE specific structure (typically the one preferring "i unpaired" first). Other valid optimal structures exist — they are equally good base-pair-wise but differ in which positions pair.

Why Nussinov is not used in practice

The textbook itself ("Evaluation of Maximizing Basepairs") lists why this is a simplistic approach:

  • Does not give accurate structure predictions.
  • Misses nearest-neighbor interactions.
  • Misses stacking interactions (stacked pairs are much more stable than isolated ones).
  • Misses loop-length preferences (hairpin loops have an optimal length around 4-8 nt; very short or very long loops are penalized).

That's why energy minimization (MFOLD, ViennaRNA) superseded Nussinov for real predictions.

PART A · 7 — Energy Minimization (MFOLD, ViennaRNA)

Better than Nussinov: instead of counting pairs, minimize the total Gibbs free energy ΔG of the predicted structure.

Key principles (from RNA_structure.pdf):
  • Thermodynamic stability — the most stable structure is the most likely.
  • Energy parameters for stacks, loops, bulges are measured experimentally.
  • Uses dynamic programming (much like Nussinov, but the score is ΔG, not pair count).
  • Energy of any position is influenced only by local sequence and structure (this is the "nearest-neighbor" model — it's what makes DP feasible).
  • Pseudoknots not handled (same DP limitation as Nussinov).
  • Implementations: MFOLD (Michael Zuker) and ViennaRNA.

Drawbacks:

  • Compute only one optimal structure (although suboptimals can be enumerated).
  • Inherits the usual problems of any purely mathematical approach — the energy model is an approximation, RNAs in cells may not be in the minimum-energy state (kinetic traps, chaperones, etc.).

PART A · 8 — Comparative Sequence Analysis & Covariance Models

The most trusted method

If you have many homologous RNA sequences, you can align them and look for compensatory mutations: a position that changes from A → G is informative only if its partner also changes from U → C, preserving the pair. Such covariation is strong evidence the two positions are base-paired in the real structure.

Tools that formalize this with stochastic context-free grammars (SCFGs): INFERNAL / Rfam, RNAalifold.

Strengths: handles pseudoknots (with appropriate models), incorporates real evolutionary signal.
Weakness: requires many homologous sequences with sufficient diversity.

PART B · 9 — Why predict protein structure?

The textbook lays out the motivation directly:

Sequence → Structure → Function. Once you know a protein's 3-D structure, you can:
  • Predict the function of novel proteins.
  • Predict the effect of mutations (disease variants).
  • Design structure-based drugs (rational drug design needs a target binding pocket).
  • Understand the protein-folding problem itself — one of biology's grand challenges.

The motivating gap

From the slides — the gap between known sequences and known structures is enormous and growing:

~250k structures vs. ~250 million sequences — three orders of magnitude. Experiments (X-ray, NMR, cryo-EM) are time-consuming, expensive, laborious. Computational prediction is the only way to close the gap.

PART B · 11 — The two governing principles (Anfinsen + Levinthal)

Anfinsen's Dogma

The native 3-D structure is determined entirely by the amino-acid sequence. The native state corresponds to the global minimum of free energy. → This is the theoretical postulate that justifies structure prediction from sequence alone.

Levinthal's Paradox

A 100-residue protein with just 3 conformational degrees of freedom per residue can theoretically adopt 3¹⁰⁰ ≈ 5×10⁴⁷ conformations. Even a supercomputer would take years to enumerate them all. Yet real proteins fold in milliseconds to microseconds. Therefore, folding cannot be a random search — it must follow a directed pathway (folding funnel). This is why naïve "try every possibility" computational approaches fail.

Together: Anfinsen says the answer exists; Levinthal says naïve search will never find it. Hence the field of protein structure prediction.

PART B · 12 — Secondary Structure Prediction — overview

Three main secondary-structure states (3-state prediction):

  • H — α-helix
  • E — β-sheet (extended strand)
  • C — coil / loop

Why some amino acids prefer some structures (amino-acid propensities)

The 20 amino acids differ in size, shape, and charge; this makes them prefer one secondary-structure context over another. From the slides:

  • Proline has a rigid, ring-locked side chain — it cannot fit into an α-helix backbone geometry and is famously a "helix breaker".
  • Glycine has no side chain — it's too flexible, also a helix-breaker (and turn-favoring).
  • Glutamic acid, methionine, alanine, leucine — strong helix-formers.
  • Valine, isoleucine, tyrosine, phenylalanine, tryptophan — strong sheet-formers.

"Classic methods" listed by the slides: Chou-Fasman and GOR. They both start from these propensities computed from databases of known structures.

PART B · 13 — ⭐ Chou-Fasman Method (deep dive)

Developed by Peter Y. Chou and Gerald D. Fasman in 1974. It's the founding statistical method of secondary-structure prediction — simple enough to do by hand, which is exactly why it appears every year on the exam.

13.1 — The core idea: conformational parameters (propensities)

For every amino acid, compute three numbers from a database of known protein structures:

  • Pα — propensity for α-helix
  • Pβ — propensity for β-sheet
  • Pturn — propensity for β-turn
Pi,j = (frequency of amino acid i in secondary structure j) / (overall frequency of amino acid i)

Interpretation:

  • P > 1 → the amino acid is over-represented in that structure → it favors that structure.
  • P = 1 → no preference (random).
  • P < 1 → under-represented → it disfavors that structure.

Worked example of the propensity calculation (from the slides)

Scenario: A database of 50 proteins has 20,000 amino acids total. Of these, 5,000 form α-helix. The database contains 2,000 alanine residues, and 500 of them are in helices.

Compute PAla, helix:
P = (frequency of Ala in helix) / (overall frequency of Ala)
  = (500 / 5,000) / (2,000 / 20,000)
  = 0.10 / 0.10
  = 1.0   → no preference (Ala is equally likely to be in helix as random)

13.3 — The Algorithm (the procedure you'll do on the exam)

A. Predicting α-helices

Step 1 — Nucleation (find a seed region)

  • Slide a 6-residue window across the sequence.
  • If a window contains at least 4 residues with P(α) > 1 (they need NOT be contiguous within the window), mark that window as a nucleation site.

Step 2 — Extension (grow the helix in both directions)

  • Try to add the next residue (forward, toward C-term; or backward, toward N-term).
  • After adding, compute the average P(α) of the 4-residue tetrapeptide at the new edge.
  • If avg P(α) ≥ 1, accept the new residue and continue extending.
  • Terminate when avg P(α) < 1. The last accepted residue is the helix boundary.

Step 3 — Resolve conflicts

  • If a region is predicted as both helix and sheet, compute avg P(α) and avg P(β) over the conflicting region.
  • Assign the structure with the higher average.

B. Predicting β-sheets — the parameters change slightly

Stepα-Helixβ-Sheet
Nucleation window size6 residues5 residues
Threshold≥ 4 residues with P(α) > 1≥ 3 residues with P(β) > 1

TerminationAvg P(α) < 1 → stopAvg P(β) < 1 → stop

C. Predicting β-turns (hairpin loops)

Turns are tighter — they involve only 4 residues, and Chou-Fasman uses position-specific frequencies f(i), f(i+1), f(i+2), f(i+3) (the last four columns of the parameter table).

p(t) = f(i) × f(i+1) × f(i+2) × f(i+3)

Predict a β-turn starting at position i if all three conditions hold:

  • p(t) > 0.000075  positional product
  • Avg P(turn) of the tetrapeptide > 1, AND
  • Avg P(turn) > Avg P(α) AND Avg P(turn) > Avg P(β) over the same tetrapeptide.

About the "logic of a 6-residue window" — PYP Q4b (1 mark)

"What is the logic behind using a six residue sliding window during the Chou-Fasman method?"

Answer: One full turn of an α-helix contains 3.6 residues. A 6-residue window therefore spans approximately 1.6 turns, which is the minimum stretch that can form a stable helical segment with hydrogen bonds between residue i and residue i+4. A shorter window would risk over-predicting tiny non-helical fluctuations; a longer one would miss short genuine helices. Six is the empirical sweet spot — long enough for at least one full helical H-bond pattern to exist, short enough to retain local resolution.

13.6 — Common Chou-Fasman exam traps

⚠ Traps and tips

  • P-values in the printed table are × 100. Don't accidentally compare 142 to 1 — divide by 100 first.
  • "Need not be contiguous" — in the nucleation window, the residues with P > 1 can be scattered. You just need a count of how many exceed 1.
  • Extension uses a tetrapeptide AT THE NEW EDGE, not the entire growing helix. Only the last 4 residues including the newly-added one are averaged.
  • Termination is strict: avg P < 1 (not "≤ 1") terminates. avg = 1.0 includes.
  • Proline kills helices. Wherever Pro sits, the tetrapeptide containing it usually drops below 1 because P(α) = 0.57. Use Pro as a quick visual "boundary marker".
  • Glycine is also P(α) = 0.57. Same effect.
  • Resolve conflicts only when you predicted overlapping helix AND sheet — then compute the average of each and assign the higher one.
  • If asked for β-sheet: remember the parameters change — window = 5, threshold = 3 residues with P(β) > 1. Same extension rule.

PART B · 14 — GOR Method (brief)

GOR (Garnier-Osguthorpe-Robson) is the second classic method. It improves on Chou-Fasman by using information theory / Bayesian inference — it considers not just the residue at position i, but the influence of neighbouring residues on the structure at i (typically a window of ±8 residues). Multiple implementations exist; GOR V (2002) is one. From the slides, GOR appears as one of nine algorithms used to predict SS for the 2SOD protein.

PART B · 16 — Tertiary Structure Prediction: Homology Modeling

Now we move from secondary (local helix/sheet) to tertiary (full 3-D atomic positions). The three approaches:

ApproachWhen it worksIdea
Homology modelingA close evolutionary relative with a known structure existsCopy the relative's structure as a template
Threading (fold recognition)No close homolog, but the structure type already exists in PDB"Thread" the target sequence onto many known folds; pick the best fitting one

16.1 — Homology Modeling: the 6-step pipeline

From protein_structure_prediction_modeling_SB.pdf:

  1. Template Identification — search the PDB. >50% sequence identity is ideal; <30% is risky.
  2. Sequence Alignment — align target to template(s). The single most critical step; alignment errors propagate to every later step.
  3. Backbone Construction — copy the backbone (Cα + main chain) from the template. Structurally Conserved Regions (SCRs) come directly across.
  4. Loop Modeling — handle insertions/deletions and Structurally Variable Regions (SVRs). The least reliable part of the model.
  5. Side-Chain Modeling — assign side-chain conformations (torsion angles χ), avoid steric clashes. Often uses rotamer libraries.
  6. Refinement & Validation — energy minimization, stereochemistry checks (Ramachandran plot), model-quality assessment (DOPE, MolProbity, ProQ, etc.). If quality is bad, iterate from earlier steps.

The pipeline ends in: target sequence → template selection → alignment → build → refine → evaluate → if good, use model; if bad, re-iterate.

16.2 — Limitations of homology modeling

  • Needs a template — no template means no model.
  • Alignment errors propagate everywhere downstream.
  • Poor loop prediction is a chronic problem.
  • Fails for novel folds — by definition.

PART B · 17 — Threading & Ab Initio

Threading (Fold Recognition)

Instead of relying on sequence similarity to find a template, threading relies on structural compatibility. The target sequence is computationally threaded onto every known 3-D fold in a library, and each alignment is scored energetically/statistically. The best-scoring fold becomes the predicted structure.

Useful when the target has < 30% identity to any PDB entry but adopts a fold already represented in PDB.

Ab initio

From scratch — no template, no fold library. Uses physics-based force fields (molecular dynamics), or statistical potentials, often with fragment libraries (Rosetta). Computationally expensive; usually limited to small proteins (< ~100-150 residues).

PART B · 18 — AlphaFold & CASP

CASP — Critical Assessment of techniques for protein Structure Prediction

  • Community-wide blind competition held every two years.
  • Research groups submit predicted structures before the experimental structures are released.
  • The CASP rankings show the latest state-of-the-art in structure prediction.
  • CASP14 (2020) — AlphaFold, developed by Google DeepMind, achieved near-experimental accuracy and changed the history of structure prediction.

AlphaFold idea (high level)

  • Inputs: target sequence → genetic-database search to build a deep MSA → structure-database search for templates.
  • Builds two representations: an "MSA representation" (single repr.) and a "Pair representation" (between residue pairs).
  • The Evoformer (48 transformer-style blocks) refines both representations using co-evolutionary information.
  • The Structure module (8 blocks) converts these into 3-D atomic coordinates.
  • Recycled three times to refine.
  • Uses spatial + (co-)evolutionary information to predict residue-residue interactions and then build the 3-D structure.

Quick-fire summary card

🧬 Nussinov in one breath

"Given an RNA sequence, find the non-crossing set of base pairs that maximizes the number of pairs. Uses dynamic programming in O(L³) time and O(L²) space. Recursion has four cases: i unpaired, j unpaired, i-j paired (+1), or bifurcation. Initialization γ(i,i)=γ(i,i-1)=0. After fill stage, γ(1,L) is the answer; structure is recovered by stack-based traceback. Cannot handle pseudoknots; ignores stacking, loop length, and thermodynamics."

🧪 Chou-Fasman in one breath

"Use empirical propensities P(α), P(β), P(turn) of each amino acid to predict secondary structure. For α-helix: slide a 6-residue window; if ≥4 residues have P(α)>1, nucleate; extend by tetrapeptide average. For β-sheet: window of 5, threshold 3. For turn: position-specific f-values plus P(turn) checks. Resolve conflicts by comparing averages. Accuracy ~60-65% — superseded by neural-network methods but still the canonical pen-and-paper algorithm."

🎯 Final exam strategy

  1. For any Nussinov question: (1) state the rules, (2) write the recursion, (3) build the DP table diagonal by diagonal, (4) traceback with the stack, (5) draw the structure and give the dot-bracket. Marks are awarded at each step — don't skip even if the score is obvious.
  2. For any Chou-Fasman question: (1) tabulate P(α) (and P(β) if asked) for every residue, (2) scan all windows and identify nucleations, (3) for each nucleation extend in both directions showing the tetrapeptide averages, (4) merge overlaps, (5) resolve any α/β conflicts. Always show the arithmetic — do not just write the final answer.
  3. Always include a one-line biological sanity check at the end (e.g., "Pro at position 5 is consistent with the helix starting at position 7", "the predicted hairpin loop has 3 unpaired bases, which is borderline acceptable", etc.). It costs no marks if wrong but signals understanding when right.
