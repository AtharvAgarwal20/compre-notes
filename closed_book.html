1️⃣ Central Dogma & Molecular Biology Basics

Core flow: DNA → (replication, self) ; DNA → RNA (transcription) ; RNA → Protein (translation) → Functional protein (folding/assembly/targeting). Information transfer from protein → nucleic acid is impossible (Crick, 1958).
PropertyProkaryotesEukaryotes

Start codonAUG, also GUG/UUG (bacteria)AUG only
Introns/SplicingAbsent (typically)Present; spliceosome (~150 proteins + 5 RNAs)
Promoter−35 / −10 (Pribnow/TATA-like, AT-rich)TATA, CAAT, GC box; enhancers (distal)

Key facts:
  • Genetic code: 3-letter codons from 4 bases → 64 codons → 20 AAs ⇒ degenerate (synonymous mostly at 3rd position — wobble).
  • Stop codons: UAA, UAG, UGA.
  • Translation components: mRNA + tRNA + aminoacyl-tRNA synthetase + ribosome.
  • tRNA features: Acceptor stem (AA attaches), TΨC loop (pseudouridine), D loop (dihydrouridine), Anticodon loop.
  • Alternative splicing: ~20,000 human genes → >100,000 protein types; ~60% of human genes alternatively spliced.

DNA Structure

  • Double-stranded, antiparallel (5′→3′ and 3′→5′).
  • Watson–Crick pairs: A=T (2 H-bonds), G≡C (3 H-bonds).
  • Purines: A, G. Pyrimidines: C, T (U in RNA).
  • Forms: B-DNA (most common, right-handed), A-DNA (dehydrated, right-handed), Z-DNA (left-handed, zig-zag).
  • Sequence in FASTA format starts with > header line.

Why align? To detect homology, infer function, build phylogenies, identify motifs/domains, predict structure, design primers.

3️⃣ Dot Matrix / Dot Plot

A 2D grid that puts a dot wherever residues of two sequences match. Visual method. No score, no quantitative output.

Interpretations (memorise — favourite MCQ source)

PatternMeaning
Main diagonal (top-left to bottom-right)Sequences are similar / identical regions
Off-diagonal parallel linesInternal repeats
Shifted diagonal (broken)Insertion or deletion (mutation)
Anti-diagonal (perpendicular to main)Inverted repeat / palindrome (when comparing seq vs reverse-complement)
Scattered isolated dotsRandom matches (noise)
Window + Stringency: Slide a window of w residues; put a dot only if ≥ stringency matches in that window. Higher window/stringency → less noise but may miss weak similarities.

4️⃣ Dynamic Programming — Needleman–Wunsch & Smith–Waterman

Bellman's principle: The best alignment up to (i,j) is the previous best + the local choice (match/mismatch/gap). Time complexity: O(m·n); Space: O(m·n). Bellman invented DP in the 1950s; Needleman & Wunsch first applied it to biological sequences in 1970.

Needleman–Wunsch (Global) — Algorithm

  1. Initialization: Create (m+1)×(n+1) matrix. F(0,0)=0; first row F(0,j) = j·gap; first column F(i,0) = i·gap.
  2. Matrix fill (recurrence): 
    F(i,j) = max {
   F(i−1, j−1) + s(xi, yj)   // diagonal: match/mismatch
   F(i−1, j)   + gap         // up:   gap in seq Y
   F(i,   j−1) + gap         // left: gap in seq X
}
    Record arrow pointing to which cell gave the max.
  3. Traceback: Start at bottom-right F(m,n) and follow arrows back to F(0,0). Diagonal → match/mismatch; up → gap in Y (insertion in X); left → gap in X.

Smith–Waterman (Local) — Algorithm

  1. Initialization: First row and column all zeros.
  2. Recurrence (extra zero term): 
    H(i,j) = max {
   0,                                  // <-- KEY DIFFERENCE
   H(i−1, j−1) + s(xi, yj),
   H(i−1, j)   + gap,
   H(i,   j−1) + gap
}
  3. Traceback: Start at the cell with the maximum value anywhere in the matrix, follow arrows until you hit a 0.
⚠ MCQ traps:
  • NW does NOT zero out negatives; SW does.
  • SW traceback can start anywhere; NW must start at F(m,n).
  • Both use the same recurrence apart from the extra "0" in SW.
  • Both are O(m·n) — same complexity.
  • BOTH guarantee the optimal alignment (under the given scoring scheme). FASTA/BLAST are heuristic and don't guarantee optimality.

5️⃣ Scoring Matrices — PAM & BLOSUM

Types of substitution matrices

  1. Identity scoring — match=1, mismatch=0. Simplest. Rarely used for proteins.
  2. DNA scoring — distinguishes transitions (A↔G, C↔T; purine↔purine, pyrimidine↔pyrimidine; more frequent) from transversions (purine↔pyrimidine; less frequent).
  3. Chemical similarity — weighted by size, charge, polarity.
  4. Observed matrices — empirical (PAM, BLOSUM). Default for proteins.

PAM (Point Accepted Mutation) — Dayhoff, 1978

  • Derived from global alignments of closely related (≥85% identity) proteins.
  • PAM1 = 1% accepted mutations per 100 residues.
  • Higher PAM numbers obtained by matrix multiplication (PAM1)ⁿ. Assumes Markov process.
  • Errors in PAM1 are magnified through PAM250.
  • Assumptions: P(X→Y) = P(Y→X); independence of sites; uniform mutation rate; AA frequencies stationary over evolutionary time.

BLOSUM (BLOcks SUbstitution Matrix) — Henikoff & Henikoff, 1992

  • Derived from local alignments — conserved blocks (ungapped).
  • BLOSUM62: sequences clustered at 62% identity. Default in BLAST.
  • Each BLOSUM matrix is constructed independently (no extrapolation).
  • Score = log-odds: Sij = 2·log₂(pij / (pi·pj)) (half-bits).
The Golden Inverse Rule (high-yield MCQ!):
Sequence relationshipBest PAMBest BLOSUM
Closely related (high identity, short)PAM1 (low)BLOSUM100 (high)
Distantly related (low identity, long)PAM250 (high)BLOSUM30 (low)
Mnemonic: PAM = Proportional, BLOSUM = Backwards. ↑PAM number = ↑distance; ↑BLOSUM number = ↓distance.

BLOSUM vs PAM — Conceptual differences

Why log-odds? (✓ multiplication → addition; ✓ scores additive; ✓ positive = favoured substitution; ✓ negative = unfavourable; ✓ zero = random expectation.) All BLAST statistics derive from log-odds scores.

6️⃣ Gap Penalties

TypeFormulaExample (gap len 4)Comment
Linearw(k) = g·kg = −2 → 4·(−2) = −8Simple. Penalizes long gaps too harshly.
Affinew(k) = wo + (k−1)·wewo=−3, we=−1 → −3 + 3·(−1) = −6Biologically realistic (single insertion of multiple residues more likely than several separate gaps).
Choosing gap penalty: For closely related sequences → higher (stricter) gap penalty (gaps rare). For divergent → lower (more tolerant). Affine always preferred biologically.

7️⃣ Statistical Significance — E-value & Karlin–Altschul

Karlin–Altschul equation: 
E = K · m · n · e^(−λ·S)
where E = expected number of alignments with score ≥ S by chance; m, n = sequence/database lengths; K, λ = matrix-specific constants (correction for edge effects, scoring matrix); S = raw score.
  • Higher S → lower E-value.
  • Random alignment scores follow the Gumbel Extreme Value Distribution (EVD), NOT a normal distribution, because we take maximum values. EVD has a long right tail.
  • PDF of EVD: f(x) = e−x − e−x.
  • E-value is the expected count of hits ≥ S. Counts of rare independent events follow a Poisson distribution → P-value derives from this.
  • p-value from E: P(X≥1) = 1 − e−E. For small E, p ≈ E.

Bit-score (universal unit)

S′ (bits) = (λ·S − ln K) / ln 2
Every +1 bit → probability of statistically significant alignment doubles (or E-value halves). Bit scores are matrix-independent, allowing comparison across databases.

Interpretation of E-values

E-valueMeaning
10~10 random hits — meaningless
1~1 random hit — possibly meaningful
0.01Rare by chance — likely meaningful
1e-20Extremely unlikely — strong homology
0Effectively certain homology
⚠ E-value depends on database size! Same alignment in a larger DB has a larger E-value (more chances to find it by random). Lower E is better.

8️⃣ Multiple Sequence Alignment (MSA)

Why MSA?

  • Conserved motif/domain discovery.
  • Phylogenetic analysis.
  • Secondary structure & 3D model (homology modeling) prediction.
  • Profile / PSSM / HMM building.

Methods

MethodApproachTools
Dynamic programming (K-dim)Exact, scales to K sequences — O(L^K). Practically infeasible for K>3–5.MSA (Lipman)
Progressive(1) Pairwise alignments → distance matrix → (2) guide tree → (3) align in tree order, sequence→sequence, sequence→profile, profile→profile.ClustalW, MUSCLE, Kalign
Iterative (consistency)Remove each sequence, realign to profile, iterate until convergence.MUSCLE, T-Coffee, DIALIGN
HMM-basedProbabilistic profile HMM.HMMER, SAM
Problems with progressive (ClustalW) alignment:
  • Errors made early propagate and cannot be corrected later ("Once a gap, always a gap").
  • Tries global, not local — bad for sequences sharing only a domain.
  • Sensitive to choice of substitution matrix and gap penalty at every stage.
  • Performs poorly when one sequence has a large insertion relative to others.

SP (Sum-of-Pairs) score

Most common MSA scoring scheme: sum scores of all pairwise alignments in the MSA column-by-column. Used in DP-based and progressive methods.

Rooted vs unrooted

  • Rooted: shows direction of evolution, has a common ancestor. Required to make ancestor–descendant statements.
  • Unrooted: only specifies relationships, not evolutionary direction.
  • An unrooted 4-taxon tree can be rooted in 5 different positions (= number of branches) → 5 distinct rooted trees.

Tree-building methods

Distance-basedCharacter-based
DataCompressed to pairwise distance matrix (info loss)Uses each alignment column directly (no compression)
SpeedFastSlow (especially MP, ML)
AccuracyLowerHigher
ExamplesUPGMA, Neighbor-Joining (NJ)Maximum Parsimony (MP), Maximum Likelihood (ML), Bayesian

UPGMA (Unweighted Pair Group Method with Arithmetic mean)

  • Assumes a constant molecular clock — all branches evolve at equal rate.
  • Output: rooted, ultrametric tree.
  • Steps: (1) each taxon = cluster; (2) join the 2 closest; (3) recompute distance to new cluster = arithmetic mean; (4) repeat.
  • Drawbacks: assumes constant rate (often violated); irreversible clustering; inaccurate.

Neighbor-Joining (NJ) — Saitou & Nei

  • Allows unequal evolutionary rates across branches.
  • Output: unrooted tree (rooting needs an outgroup).
  • Uses corrected distances (averages to other leaves) — not just nearest neighbour.
  • Generally more accurate than UPGMA.

Maximum Parsimony (MP)

  • Principle: prefer the tree requiring the fewest evolutionary changes (Occam's razor).
  • Assumes: evolution is rare; events independent across lineages.
  • Best for strong sequence similarity; computationally expensive; fails when variation is high.
Choosing method (flowchart):
  1. Strong sequence similarity → Maximum Parsimony
  2. Clearly recognisable similarity → Distance methods (NJ)
  3. Weak / no recognisable similarity → Maximum Likelihood

🔟 Maximum Likelihood & Bootstrapping

Maximum Likelihood (ML) for phylogeny

Likelihood = P(Data | Model). Goal: find parameter values that maximise this probability.
Parameters: tree topology, branch lengths, nucleotide frequencies, substitution rates.
Assumptions: each column is an independent observation; columns contain homologous bases.
  • Iterative: start with random parameters → adjust to improve likelihood → repeat until peak.
  • May get stuck at local maxima — does NOT guarantee global maximum.
  • Output: ML estimate of topology, branch lengths, other parameters, plus a likelihood value (model fit).

Bootstrapping (tree reliability)

  1. Randomly resample columns of the alignment with replacement (not sequences!).
  2. Build a tree from each pseudo-alignment.
  3. Repeat N times (N = 100, 500, 1000, 10000).
  4. Bootstrap value of a clade = % of replicates in which the clade appears.
Bootstrap valueInterpretation
> 90%Strong support
70–90%Moderate support
< 70%Weak / unreliable
What bootstrap means (MCQ trap):
  • ✓ Stability of grouping under sampling variation.
  • ✗ NOT the probability that the group is "true".
  • ✗ NOT confidence in the evolutionary model itself.

1️⃣1️⃣ Gene Prediction

What's predicted?

  • Coding region; encoding strand; gene start/end; exon-intron boundaries (in eukaryotes); regulatory sequences.

Three categories

TypeIdeaStrengths
Ab initio / IntrinsicUses only the DNA sequence; detects statistical patterns (codon usage, hexamer frequencies, GC content, ORFs)Works on novel genomes; no DB needed
Homology / ExtrinsicAligns genome to known gene/protein databasesHigh specificity
Hybrid / ComparativeCombines ab initio + extrinsic evidence (e.g. RNA-seq evidence guiding HMMs)Best overall accuracy

Features used

Signal features (discrete markers)Content features (statistical, region-spread)
Promoter (−35, −10 / TATA box); Start codon (ATG); Stop codons (TAA, TAG, TGA); Splice sites (donor GT, acceptor AG)Codon usage bias; hexamer frequencies; ORF length; GC content / CpG islands

ORF-based gene prediction

  • Scan DNA in all 6 reading frames (3 forward, 3 reverse).
  • Find longest ORFs (start → stop).
  • Filter by min length (typically >100 codons).
  • Limitations: poor in eukaryotes (long introns); random long ORFs possible; misses short genes; ignores regulatory signals.

Markov Models

  • Two separate models: coding vs non-coding. Compute P(sequence | model) for each; predict class by higher likelihood.
  • Limitation: can't explicitly model gene structure (no exon/intron boundaries).

Hidden Markov Models (HMMs)

Extends MM by introducing hidden states (exon, intron, intergenic, start codon, stop codon).
Transition probabilities — likelihood of moving between states (e.g., exon → intron).
Emission probabilities — probability of a state emitting a specific base.
Viterbi algorithm: finds the most likely sequence of hidden states given an observation.
Training: Maximum Likelihood Estimation.
HMM is a machine learning model (sequence labelling task). Tools using HMM-based gene prediction: GENSCAN, AUGUSTUS, GeneMark.

1️⃣2️⃣ Protein Secondary Structure Prediction — Chou-Fasman

Levels of protein structure

TertiaryInteractions of secondary structures (full 3D fold)Hydrophobic effect (+ H-bonds, salt bridges, vdW, disulfides)

Analogy: alphabets → words → sentences → paragraphs.

Chou-Fasman conformational parameters

Pij = (frequency of AA i in SS j) / (overall frequency of AA i).
Example: 50 proteins, 20,000 AAs total, 5,000 form helix. Of 2,000 Ala residues, 500 form helix.
P(Ala, helix) = (500/5000) / (2000/20000) = 0.1 / 0.1 = 1.0.

Homology modeling steps (Sali, MODELLER)

  1. Template identification — BLAST against PDB.
  2. Sequence alignment — most critical step; errors propagate to the model.
  3. Backbone generation — copy coordinates from template.
  4. Loop modeling — handle insertions/deletions; Structurally Variable Regions (SVRs); least reliable regions.
  5. Side-chain modeling — assign χ angles; avoid steric clashes.
  6. Evaluation — Ramachandran plot, energy, ligand fit.
AlphaFold (DeepMind, EMBL-EBI) — Uses MSA + (co-)evolutionary information; predicts residue–residue contacts; constructs 3D models with very high accuracy. AlphaFold DB has 200+ million predicted structures. PDB has ~252,842 experimental structures vs ~1,068,577 Computed Structure Models. Drastically reduced the sequence–structure gap.

1️⃣4️⃣ RNA Secondary Structure — Nussinov Algorithm

RNA structural elements

ElementDescription
Stem (helix)Continuous base-paired region
Hairpin loopUnpaired bases at the end of a stem (typically ≥4 nt)
Bulge loopUnpaired bases on ONE strand interrupting a stem
Interior loopUnpaired bases on BOTH strands
Junction (multiloop)≥3 stems converging
PseudoknotCrossed base pairs — NOT predicted by standard Nussinov / Zuker

Base pairing rules

  • Watson-Crick: A=U, G≡C.
  • Wobble: G·U (allowed in RNA; weaker).
  • A nucleotide can form at most one pair.
  • i and j are pairable only if |i − j| ≥ 4 (no immediate neighbours, hairpin loop must be ≥3 nt).
  • No crossings allowed (no pseudoknots in basic Nussinov): if (i,j) and (i′,j′) are pairs and i < i′, then must have i < i′ < j′ < j.

Nussinov recursion (Fill stage)

γ(i, j) = max {
   γ(i+1, j),                          // i unpaired
   γ(i, j−1),                          // j unpaired
   γ(i+1, j−1) + δ(i, j),              // i pairs with j
   max over k (i<k<j) [γ(i,k) + γ(k+1,j)]  // bifurcation
}
where δ(i,j) = 1 if (xi, xj) is a valid pair, else 0.
Initialization: γ(i,i) = 0; γ(i+1,i) = 0.
Score (max pairs) is at γ(1, L) (top-right of the upper-triangle matrix).

Traceback

Start at γ(1, L); at each step, determine which of the 4 cases gave the maximum and recurse on those subsequences. Records base pairs forming the secondary structure.
Nussinov limitations:
  • Maximises base pairs only — ignores stacking, loop length penalties, nearest-neighbour thermodynamics.
  • Cannot predict pseudoknots.
  • For realistic free-energy prediction → use Zuker's algorithm (Mfold, RNAfold) with thermodynamic parameters.

1️⃣6️⃣ Microarray Technology

Purpose: Quantitative measurement of gene expression — thousands of genes in parallel.

Two-channel (cDNA) microarray workflow

  1. Obtain cells → extract mRNA.
  2. Reverse-transcribe to cDNA; label two samples with different fluorophores: Cy3 (green) for control, Cy5 (red) for sample.
  3. Mix labelled cDNAs; hybridise to microarray chip (each spot = probe of one gene's DNA).
  4. Wash unbound cDNA; laser-scan to detect fluorescence.

Spot colour interpretation

ColourMeaning
Green (only Cy3)High in control
Red (only Cy5)High in sample
Yellow (both)Equally expressed
Black (neither)Not expressed

Microarray data analysis types

  • Differential expression — fold-change analysis.
  • Classification (supervised) — identify disease, predict outcome.
  • Clustering (unsupervised) — discover new classes (hierarchical, k-means).

Traditional methods microarray replaces

Northern blotting; RT-PCR / qRT-PCR — single gene at a time.

Shotgun sequencing

  • Cut genome into short fragments → sequence each → assemble via overlaps → contigs.
  • First applied to whole-genome sequencing in 1995 (Haemophilus influenzae).
  • Standard for prokaryotic genome sequencing; used for Drosophila & human.

1️⃣8️⃣ ChIP-seq & Epigenomics

ChIP-seq = Chromatin Immunoprecipitation + NGS. Identifies DNA regions bound by a protein (transcription factor, modified histone).

Workflow

  1. Crosslink DNA-protein in vivo.
  2. Shear chromatin.
  3. Immunoprecipitate with antibody against TF or histone mark.
  4. Reverse crosslink → purify DNA.
  5. End-repair, adaptor ligation, NGS.
  6. Map reads to reference genome; identify peaks/motifs.

Common histone marks

  • H3K4me3 — Histone H3 Lysine 4 tri-methylation → active promoters.
  • H3K9me2/3 — Lysine 9 di-/tri-methylation → heterochromatin / silencing.
  • H3K27me3 — Lysine 27 tri-methylation → Polycomb repression.

📝 PART B — MOCK PAPER 1 (20 MCQs)

Q1. Which of the following statements about the Smith-Waterman algorithm are TRUE?
  • The first row and first column of the DP matrix are initialised with zeros.
  • Negative scores in the matrix are replaced with zero.
  • The traceback always begins at the bottom-right corner.
  • It guarantees the optimal local alignment for a given scoring scheme.
Correct: A, B, D
C is wrong — SW traceback starts at the cell with the maximum value, not bottom-right (that's NW). SW is DP, so it guarantees optimality (unlike heuristics like BLAST/FASTA).
Q2. Which scoring matrices would be most appropriate for aligning two distantly related proteins?
  • PAM1
  • PAM250
  • BLOSUM30
  • BLOSUM100
Correct: B, C
Remember the inverse rule. Distant = high PAM (250) and low BLOSUM (30). For closely related: PAM1 and BLOSUM100.
Q3. The central dogma of molecular biology (Crick, 1958) holds that:
  • Information cannot be transferred from protein back to nucleic acid.
  • RNA can be reverse-transcribed into DNA.
  • Information transfer from protein to protein is permitted.
  • Once information has passed into protein, it cannot get out again.
Correct: A, D
B was discovered later (reverse transcriptase, retroviruses) and is consistent with the dogma. C is explicitly forbidden by Crick's statement.
Q4. Which statements about dot-matrix analysis are CORRECT?

  • A perpendicular (anti-diagonal) line indicates an inverted repeat / palindrome.
  • The method outputs a numerical alignment score.
  • Increasing window size and stringency reduces noise.
Correct: A, B, D
C is wrong — dot plot is purely visual; it has no numerical score, alignment, or trace.
Q5. The Karlin–Altschul equation E = K·m·n·e−λS has which properties?
  • E-value increases when the database size (n) is larger.
  • Higher raw scores S yield exponentially lower E-values.
  • The constants K and λ depend on the scoring matrix.
  • It assumes random alignment scores follow a normal distribution.
Correct: A, B, C
D is wrong — random alignment scores follow the Gumbel Extreme Value Distribution (EVD), not normal — because we take maxima.
Q6. About UPGMA:
  • It assumes a constant molecular clock.
  • It produces an unrooted tree.
  • Distance updates use the arithmetic mean.
  • It can correct for unequal evolutionary rates among lineages.
Correct: A, C
UPGMA gives a rooted, ultrametric tree (B false). It cannot handle unequal rates — that's Neighbor-Joining (D false).
Q7. Bootstrap analysis in phylogenetics involves:
  • Sampling alignment columns with replacement to make pseudo-alignments.

  • Bootstrap values >70% are universally accepted as strong support.
Correct: A, C
B is wrong — bootstrap is a measure of sampling stability, NOT truth probability. D is wrong — typically >90% strong, 70–90% moderate, <70% weak.
Q8. Which are TRUE about Chou-Fasman α-helix prediction?

Q9. The Nussinov algorithm:

  • Uses dynamic programming with O(L³) time complexity.

B wrong — Nussinov forbids crossings (no pseudoknots). C wrong — stacking energies are added in Zuker's algorithm, not Nussinov.
Q10. Sanger dideoxy sequencing requires:

Q11. Which are characteristics of Illumina (Solexa) sequencing?

Q15. Which are TRUE about gap penalties?

Q1. A linear gap penalty of −2 per gap and a gap of length 4 gives a total penalty of:

  • 7

Q4. In a database of 100 proteins (20,000 AA total), 5,000 form α-helix. Of 1,000 Leucine residues, 400 form α-helix. The Chou-Fasman P(Leu, α) =

  • Exponential in n

🪤 PART E — Top 30 Trap Statements (Memorise these distinctions!)

For multi-correct exams, examiners love subtle reversals. This list is your last-minute insurance.
  1. TRAP Higher PAM number = MORE evolutionary distance; higher BLOSUM number = LESS distance. The relationship is inverse.
  2. TRAP Homology is NOT a percentage — sequences are either homologous or not. Only similarity/identity are measured in %.
  3. TRAP Analogs share function but NOT ancestry — they are NOT homologs.
  4. TRAP Dot plot does NOT produce a quantitative score or alignment — only a visual pattern.
  5. TRAP Smith-Waterman traceback starts at the MAXIMUM score, not the bottom-right. Needleman-Wunsch starts at bottom-right.
  6. TRAP Both NW and SW guarantee optimality (under the scoring scheme) — BLAST and FASTA do NOT (they are heuristic).
  7. TRAP In Chou-Fasman α-helix nucleation, the 4 (out of 6) residues with Pα>1 need NOT be contiguous.
  8. TRAP The α-helix window is 6 AA; the β-sheet window is 5 AA. Extension window in BOTH is 4.
  9. TRAP Proline = helix breaker (rigid side chain); Glycine = helix breaker (too flexible). Both are excellent turn formers.
  10. TRAP Random alignment scores follow Gumbel EVD, NOT normal distribution.
  11. TRAP Bootstrap value is NOT the probability that the clade is "true" — it's the stability under resampling.
  12. TRAP Bootstrap resamples COLUMNS of the alignment with replacement, NOT sequences.
  13. TRAP UPGMA → rooted, ultrametric, constant-clock assumption. NJ → unrooted, allows unequal rates.
  14. TRAP An unrooted tree does NOT specify the direction of evolution.
  15. TRAP Nussinov does NOT predict pseudoknots. (Pseudoknots = crossing base pairs, forbidden by the algorithm.)
  16. TRAP Nussinov maximises base PAIR count only — ignores stacking and loop energies. Use Zuker for real free-energy minimisation.
  17. TRAP In Nussinov, immediate neighbours cannot pair (|i−j| ≥ 4, allowing a minimum 3-nt hairpin loop).
  18. TRAP Maximum Likelihood does NOT guarantee a global optimum — it can be stuck at local maxima.
  19. TRAP HMMs use BOTH transition AND emission probabilities.
  20. TRAP Viterbi algorithm = single most likely path. Forward algorithm = total probability summed over paths.
  21. TRAP Maxam-Gilbert uses chemical cleavage and does NOT require DNA polymerase. Sanger does.
  22. TRAP 454 (pyrosequencing) uses emulsion PCR. Illumina uses bridge PCR. SOLiD uses sequencing-by-ligation.
  23. TRAP Microarrays: green (Cy3) = control; red (Cy5) = sample; yellow = equal; black = neither.
  24. TRAP H3K4me3 = active promoter; H3K9me2/3 and H3K27me3 = repressive marks.
  25. TRAP Affine gap penalty: opening cost + (k−1) × extension cost — NOT k × extension cost.
  26. TRAP For closely related sequences, use a STRICTER (higher magnitude) gap penalty.
  27. TRAP The phase problem in X-ray crystallography means we lose the phase information of diffracted waves; intensity = amplitude² gives only amplitude.
  28. TRAP NMR upper limit ≈ 60 kDa native, ~100 kDa with isotope labelling. Cryo-EM handles very large complexes.
  29. TRAP ClustalW is matrix-based progressive; T-Coffee is consistency-based. MUSCLE iterative.
  30. TRAP The longest ORF doesn't always mean a gene — especially in eukaryotes (introns split ORFs). HMMs handle this; pure ORF scanning fails.

🔑 One-liner identifications

  • NW = global, full traceback from corner
  • SW = local, zeros in negatives, start at max
  • BLAST/FASTA = heuristic word-based
  • PAM = global, extrapolated, Markov
  • BLOSUM = local, blocks, observed directly
  • Chou-Fasman = statistical SS prediction
  • Nussinov = max base pairs RNA
  • Zuker = min free energy RNA
  • UPGMA = rooted, constant clock
  • NJ = unrooted, unequal rates
  • MP = fewest changes
  • ML = highest P(Data|Model)
  • HMM = states + transitions + emissions + Viterbi
  • 454 = emulsion PCR + pyrosequencing
  • Illumina = bridge PCR + reversible terminator SBS
  • SOLiD = sequencing by ligation
  • ChIP-seq = TF/histone binding sites genome-wide
  • cDNA microarray = Cy3/Cy5; gene expression
🎯 Exam strategy for multi-correct with negative marking:
  1. Mark only options you are certain are TRUE.
  2. Leave doubtful options BLANK — losing 0 marks is better than losing 1.
  3. Watch for "all of the above" — sometimes all 4 ARE correct (Q1 Mock 2!).
  4. Watch for subtle reversals: "more" vs "less", "rooted" vs "unrooted", "global" vs "local".
  5. Re-read the stem after each option — examiners change the verb ("includes" → "excludes").
